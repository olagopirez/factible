Los conectores

Cada organismo estatal habla un dialecto distinto — SOAP de 2012, XMLs firmados, formatos de archivo propios. factible los traduce a una API moderna y consistente.

Cómo está construido

⚡ DX primero

Tipos estrictos, quickstart en minutos y errores del Estado traducidos a mensajes útiles en español.

🧪 Simuladores incluidos

Cada conector trae un mock del organismo: testeá tu integración completa sin certificados ni trámites.

📋 Fiel a la spec

Los esquemas oficiales viven en el repo y los tests validan contra ellos en cada cambio.

🔓 Open source

MIT, autohosteable, sin lock-in. El protocolo documentado, no enterrado en una caja negra.

Destacado: facturación electrónica (CFE)

Desde 2025 todo contribuyente de IVA emite factura electrónica. @factible/cfe cubre el ciclo completo contra DGI — con una DGI simulada para desarrollar desde el día uno.

// Numeración CAE + XML v25.2 + firma XMLDSig + sello digital, en una llamada
const cfe = await factible.emitir({
  tipo: TipoCFE.E_TICKET,
  moneda: 'UYU',
  lineas: [{ cantidad: 2, descripcion: 'Café', precioUnitario: 190, indicadorFacturacion: 3 }],
});

// Enviar y entender la respuesta — sin descifrar códigos de error de un PDF
const resultado = await enviarSobreADgi(transporte, sobre);
resultado.acuseCfe.detalles[0]
// → { aceptado: false, motivosRechazo: [{ motivo: 'E05', glosa: 'Firma electrónica no es válida' }] }
⚠️ Beta offline. El pipeline completo funciona y está testeado contra los esquemas oficiales, pero aún no fue homologado contra el ambiente de testing de DGI. No usar en producción todavía.
