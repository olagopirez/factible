Validación y dígito verificador de documentos uruguayos: **CI** y **RUT**. Cero dependencias.

Es el conector con **ports oficiales a cinco lenguajes**: los cinco implementan el mismo algoritmo con los mismos vectores de test (incluido el RUT real de DGI: `219999830019`) y el CI los corre todos en cada push. Elegí tu lenguaje — la selección se mantiene en toda la página.

## API

| Operación | Qué hace |
| --- | --- |
| `validarCi(ci)` | Valida una CI completa (con dígito verificador). Acepta puntos y guión. |
| `digitoVerificadorCi(numero)` | Dígito verificador para un número de CI (sin dv, hasta 7 dígitos). |
| `formatearCi(ci)` | Formatea como `1.234.567-2`. |
| `normalizarCi(ci)` | Quita puntos, guiones y espacios. |
| `validarRut(rut)` | Valida un RUT de 12 dígitos. |
| `digitoVerificadorRut(base)` | Dígito verificador para los 11 dígitos base — `null` si el número no admite RUT válido. |
| `normalizarRut(rut)` | Quita puntos, guiones y espacios. |

Los nombres se adaptan a la convención de cada lenguaje (`validar_ci` en Python, `ValidarCI` en Go, `Validar::validarCi` en PHP, `Validar.validarCi` en Java) pero el comportamiento es idéntico.

## Por qué hay ports

Los paquetes **diminutos, de spec congelada y cero dependencias** se portan a otros lenguajes ([política multi-lenguaje](https://github.com/olagopirez/factible/blob/main/docs/plan-conectores.md)). La lógica pesada (como `@factible/cfe`) no se porta: multi-lenguaje llegará vía [gateway REST](/factible/gateway/).
